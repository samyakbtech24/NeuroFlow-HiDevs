# Task 11: Next.js Interactive Dashboard Implementation Plan

This plan covers the implementation of a full-featured frontend web application using Next.js to provide an interactive user interface for querying pipelines, inspecting system status, and analyzing RAG evaluations.

## Proposed Changes

### Setup & Infrastructure
- **Branch Strategy:** Checkout `task-10`, branch to `task-11`.
- **Framework:** Scaffold a new Next.js application inside the `frontend/` directory using `npx create-next-app@latest`.
- **Styling:** Use TailwindCSS, Framer Motion for micro-animations, and Lucide Icons for UI elements.
- **Design Philosophy:** Create a deeply immersive, premium, dark-mode aesthetic with glassmorphism and rich gradients.

---

### Core UI Architecture
- **[NEW]** Setup standard Next.js App Router layout (`app/layout.tsx` and `app/page.tsx`).
- **[NEW]** Create a centralized layout featuring a sidebar for navigation (Pipelines, Evaluations, System Health, Queries).
- **[NEW]** Implement robust API fetching wrappers inside the frontend to communicate with the FastAPI backend.

---

### Dashboard Modules
- **[NEW] Pipeline Viewer (`app/pipelines/page.tsx`):**
  - Fetch and display active AI pipelines.
  - Show aggregate evaluation scores and configured models.
- **[NEW] Query Interface (`app/query/page.tsx`):**
  - Implement a highly interactive chat-like interface.
  - Select an active pipeline and send RAG queries.
  - Render Server-Sent Events (SSE) token streams directly onto the UI in real-time.
  - Display extracted citations and document context chunks dynamically.
- **[NEW] System Health Panel (`app/health/page.tsx`):**
  - Fetch data from the `/health` endpoint.
  - Visualize the real-time status of Circuit Breakers, Queue Depths, Postgres, and Redis.
- **[NEW] Evaluation Analytics (`app/evaluations/page.tsx`):**
  - Graph and table layouts to review historical `faithfulness` and `relevance` grades generated by the LLM-as-a-judge system.

---

### Integration
- **[MODIFY]** Update `docker-compose.yml` to build and serve the `frontend` container on port 3000 alongside the backend infrastructure.
