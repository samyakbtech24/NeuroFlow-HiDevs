# NeuroFlow Render Deployment Guide

This document provides step by step instructions for deploying NeuroFlow to the Render cloud platform. Render automatically builds and deploys applications directly from a connected GitHub repository without requiring command line intervention.

## 1. Database Provisioning

Deploy the databases before deploying the application services.

1. Navigate to the Render dashboard.
2. Click "New" and select "PostgreSQL".
3. Provide a name for the database.
4. Select the appropriate region and instance size.
5. Click "Create Database".
6. Copy the "Internal Database URL" from the database settings page.
7. Click "New" again and select "Redis".
8. Provide a name for the Redis instance.
9. Click "Create Redis".
10. Copy the "Internal Redis URL" from the settings page.

## 2. API & Worker Service Deployment

The Web Service will run both the FastAPI backend and the Background Worker concurrently in a single free container.

1. Click "New" in the Render dashboard and select "Web Service".
2. Choose "Build and deploy from a Git repository".
3. Connect your GitHub account and select the repository.
4. Set the "Root Directory" to `backend`.
5. Set the "Environment" to "Docker".
6. Specify the "Dockerfile Path" as `Dockerfile`.
7. Navigate to the "Advanced" section and click "Add Environment Variable".
8. Add the variables listed in the `.env.example` file.
9. Map the `POSTGRES_URL` and `REDIS_URL` variables to the internal URLs copied earlier.
10. Click "Create Web Service".
11. Wait for the build and deployment process to complete.
12. Locate the public URL generated by Render at the top of the service page.

## 3. Production Verification Checklist

Execute these steps against your live production URL to verify the deployment.

1. Perform a GET request to `/health` and ensure it returns a green status.
2. Upload `tests/fixtures/test_doc.pdf` via POST `/ingest` and verify the document status reaches complete.
3. Submit a query about the test document via POST `/query` and ensure generation completes with a cited answer.
4. Perform a GET request to `/evaluations` and check that the entry includes scores.
5. Verify streaming functionality via GET `/query/{run_id}/stream` and ensure tokens arrive progressively.
6. Perform a GET request to the MLflow instance URL and verify experiments are visible.
7. Verify Prometheus metrics via GET `/metrics` and confirm custom metrics are present.

## 4. Rollback Procedure

If a deployment fails, follow these steps to restore the previous version.

1. Navigate to the Web Service or Background Worker in the Render dashboard.
2. Click on the "Events" tab.
3. Locate the previous successful deployment in the event history.
4. Click the "Rollback" button next to that deployment.
5. Render will automatically redeploy the exact Docker image associated with that build.
6. Verify the rollback succeeded by checking the `/health` endpoint.
7. If database migrations must be reversed, connect to the database via a local SQL client using the external database URL and execute the downgrade scripts manually.
